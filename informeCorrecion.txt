## 1. Correccion de la arquitectura por capas

### Problema identificado

Las vistas accedian directamente a objetos y colecciones del dominio mediante metodos
publicos de EmpresaTransporte. Metodos como getMyTickets(), listarSalidas(), getSalidaPorId()
exponian entidades del dominio (Salida, Ruta, PasajeTicket, Bus) hacia la capa de
presentacion, generando una violacion sistematica de la arquitectura por capas.

### Correccion realizada

Se eliminaron los getters publicos que retornaban colecciones del dominio y se hicieron
privados los metodos de busqueda interna. Se crearon 25 nuevos metodos de servicio en
EmpresaTransporte que retornan unicamente String, String[][], Object[][], int y double.
Las vistas dejaron de manipular objetos del dominio y ahora trabajan exclusivamente con
datos preparados por la capa de negocio.

## 3. Correccion de consistencia en la venta ida y vuelta

### Problema identificado

En la version anterior, la venta de viajes ida y vuelta creaba los tickets de ida antes de
verificar que los de vuelta pudieran generarse. Si la vuelta fallaba, los tickets de ida
quedaban registrados permanentemente, dejando el sistema en un estado inconsistente.

### Codigo anterior

```java
// EmpresaTransporte.ventaIdaYVuelta() -- version anterior
// Los tickets de ida se creaban ANTES de validar completamente la vuelta
PasajeTicket[] ticketsIda = crearTicketsInterno(idSalidaIda, puestosIda, pasajerosIda, true);
if (ticketsIda == null) return false;

PasajeTicket[] ticketsVuelta = crearTicketsInterno(idSalidaVuelta, puestosVuelta, pasajerosVuelta, true);
if (ticketsVuelta == null) {
    // PROBLEMA: ticketsIda ya fueron creados, asientos asignados y pasajes incrementados.
    // El sistema queda inconsistente.
    return false;
}
```

### Correccion realizada

Se implemento una validacion exhaustiva previa que verifica todas las condiciones
(disponibilidad de ruta inversa, estado PROGRAMADA, puestos consecutivos, puestos libres)
antes de realizar cualquier mutacion sobre el estado del sistema. Ademas, se agrego un
mecanismo de reversion (rollback) para casos extremos.

### Codigo corregido

```java
// EmpresaTransporte.ventaIdaYVueltaFormateada() -- version corregida
// 1. Validar existencia y estado PROGRAMADA de ambas salidas
if (salidaIda == null || salidaVuelta == null) return "Salida no encontrada";
if (!Salida.PROGRAMADA.equals(salidaIda.getEstado()))
    return "La salida de ida no esta PROGRAMADA";
if (!Salida.PROGRAMADA.equals(salidaVuelta.getEstado()))
    return "La salida de vuelta no esta PROGRAMADA";

// 2. Validar ruta inversa
if (!salidaIda.getMyRuta().getOrigen().equals(salidaVuelta.getMyRuta().getDestino())
    || !salidaIda.getMyRuta().getDestino().equals(salidaVuelta.getMyRuta().getOrigen()))
    return "La vuelta debe ser la ruta inversa";

// 3. Validar puestos consecutivos en AMBAS salidas
int[] bloqueIda = verificarPuestosConsecutivos(idSalidaIda, puestosIda.length);
int[] bloqueVuelta = verificarPuestosConsecutivos(idSalidaVuelta, puestosVuelta.length);
if (bloqueIda == null) return "No hay puestos consecutivos en la salida de ida";
if (bloqueVuelta == null) return "No hay puestos consecutivos en la salida de vuelta";

// 4. Validar que cada puesto especifico este libre
if (!todosPuestosLibres(salidaIda, puestosIda)) return "Algun puesto de ida ya esta ocupado";
if (!todosPuestosLibres(salidaVuelta, puestosVuelta)) return "Algun puesto de vuelta ya esta ocupado";

// 5. Solo despues de validar TODO: crear tickets
PasajeTicket[] ticketsIda = crearTicketsInterno(idSalidaIda, puestosIda, pasajeros, true);
PasajeTicket[] ticketsVuelta = crearTicketsInterno(idSalidaVuelta, puestosVuelta, pasajeros, true);
if (ticketsVuelta == null) {
    revertirTickets(ticketsIda); // NUEVO: revierte ida si vuelta falla
    return "Error al crear tickets de vuelta. Los tickets de ida fueron revertidos.";
}
```

### Metodo de reversion agregado

```java
// EmpresaTransporte -- nuevo metodo privado para deshacer tickets
private void revertirTickets(PasajeTicket[] tickets) {
    if (tickets == null) return;
    for (PasajeTicket t : tickets) {
        myTickets.remove(t);
        Salida s = t.getMySalida();
        if (s != null) {
            Puesto[] puestos = s.getMyPuestos();
            int num = t.getPuesto();
            if (num >= 1 && num <= puestos.length) {
                puestos[num - 1].setMyPasajero(null);
            }
        }
    }
}
```

### Resultado

La operacion ahora es atomica: si todas las validaciones previas son exitosas se crean
todos los tickets; si alguna validacion falla, no se crea ninguno.

## 4. Eliminacion de codigo duplicado

### Problema identificado

Las clases BusTipoNormal y BusTipoEjecutivo contenian implementaciones identicas del
metodo mostrarPuestosLibres(), duplicando 15 lineas de codigo en cada subclase.

### Codigo anterior

```java
// BusTipoNormal.java -- 24 lineas
public class BusTipoNormal extends Bus {
    public BusTipoNormal(String placa, String estado, int capacidad) {
        super(placa, estado, capacidad);
    }
    @Override
    public int[] mostrarPuestosLibres() {
        int count = 0;
        for (Puesto p : myPuestos) { if (p.estaLibre()) count++; }
        int[] libres = new int[count];
        int idx = 0;
        for (int i = 0; i < myPuestos.length; i++) {
            if (myPuestos[i].estaLibre()) libres[idx++] = i + 1;
        }
        return libres;
    }
}

// BusTipoEjecutivo.java -- identico, 24 lineas duplicadas
public class BusTipoEjecutivo extends Bus {
    // ... misma implementacion de mostrarPuestosLibres() ...
}
```

### Correccion realizada

La implementacion fue trasladada a la clase abstracta Bus como metodo concreto. Las
subclases heredan el metodo sin necesidad de override.

### Codigo corregido

```java
// Bus.java -- metodo concreto en la clase padre
public int[] mostrarPuestosLibres() {
    int count = 0;
    for (Puesto p : myPuestos) { if (p.estaLibre()) count++; }
    int[] libres = new int[count];
    int idx = 0;
    for (int i = 0; i < myPuestos.length; i++) {
        if (myPuestos[i].estaLibre()) libres[idx++] = i + 1;
    }
    return libres;
}

// BusTipoNormal.java -- solo 6 lineas, sin override
public class BusTipoNormal extends Bus {
    public BusTipoNormal(String placa, String estado, int capacidad) {
        super(placa, estado, capacidad);
    }
}

// BusTipoEjecutivo.java -- solo 6 lineas, sin override
public class BusTipoEjecutivo extends Bus {
    public BusTipoEjecutivo(String placa, String estado, int capacidad) {
        super(placa, estado, capacidad);
    }
}
```

### Beneficio obtenido

Se eliminaron 30 lineas de codigo duplicado. Cualquier cambio futuro en la logica de
mostrarPuestosLibres() se realiza en un solo lugar.

## Conclusiones

La retroalimentacion recibida permitio identificar aspectos importantes relacionados con
el diseno arquitectonico y la correcta aplicacion de la separacion por capas.

Las correcciones realizadas transforman el proyecto de una arquitectura donde las vistas
manipulaban directamente las entidades del dominio, a una arquitectura en capas donde
EmpresaTransporte actua como fachada (Facade) que expone unicamente los datos necesarios
para la interfaz grafica.

Esto fortalece la estructura del proyecto, reduce el acoplamiento entre componentes,
facilita el mantenimiento futuro y mejora la consistencia general del sistema.

Agradezco las observaciones realizadas, ya que me permitieron comprender mejor estos
conceptos y aplicarlos adecuadamente en la version corregida del proyecto.
